# 程序运行与参数调节说明

本文档说明 `neko` 目录下 QML 面板、Web 备用入口、模型编译和运行时推理程序的使用方式。当前主链路为 QML 面板，Web 面板仅作为备用/诊断入口保留。

## 首次使用顺序

1. 确认系统为 Windows x64，并已安装 NVIDIA 显卡驱动。
2. 双击 `1_install_vcredist.bat` 安装 VC++ 运行库。
3. 如缺少 Python 依赖，双击 `5_install_panel_python_deps.bat`。
4. 双击 `6_run_qml_panel.vbs` 启动 QML 面板。
5. 在 QML 面板中选择模型或 `.engine`。
6. 按需编译 `.engine`。
7. 调整参数并保存。
8. 点击“启动推理”。

## 面板入口

- 主入口：`6_run_qml_panel.vbs`。
- QML 启动器：`run_panel_hidden.pyw`。
- QML 应用：`gui_qml_trial.py`。
- QML 界面：`qml/Main.qml`。
- QML 后端桥：`backend/qml_bridge.py`。
- Web 备用入口：`6_run_web_panel.vbs`。
- 卡密/授权验证由推理核心启动时处理，QML 面板启动阶段不弹卡密验证。

## 核心运行链路

- 面板状态保存到 `gui_settings.json`。
- 推理核心最终读取 `runtime/config.txt`。
- 面板启动核心程序：`runtime/TRT_ZeroCopy_Pipeline.exe`。
- 输入驱动 DLL：`runtime/logi_driver.dll`，更新时默认保留。

## 更新规则

QML 是当前主链路，普通更新包不要删除以下文件：

- `qml/`
- `backend/qml_bridge.py`
- `6_run_qml_panel.vbs`
- `run_panel_hidden.pyw`
- `gui_qml_trial.py`

只有明确发布 Web-only 回滚包时，才允许把这些路径写入 `delete[]`。

## 常用参数

- `ROI`：检测区域大小。越小速度越快，越大视野越广但更容易切换目标。
- `Conf`：置信度阈值。过高容易漏检，过低容易误检。
- `NMS`：重复框抑制阈值。通常可从 `0.45` 到 `0.65` 之间尝试。
- `PID-P`：跟随强度。过低跟不上，过高容易抖。
- `PID-I`：积分项。用于消除持续小偏差，过高会拖尾。
- `PID-D`：阻尼项。用于抑制突变，过高会变慢。
- `Y 轴偏移`：控制瞄准点在框内的纵向位置。
- `卡尔曼预测`：用于移动目标提前量，过高容易前冲。
- `压枪强度`：控制开火时向下补偿速度。
- `ESP32 串口`：启用 ESP32 输出时选择 COM 口和波特率。
- `锁定类别`：只让勾选类别参与控制。类别 `0` 也是有效类别。

## 调参建议

1. 先只调 `ROI`、`Conf`、`NMS` 和目标类别，确认检测稳定。
2. 再调 `PID-P/I/D` 和 `Y 轴偏移`，让基础跟随顺滑。
3. 最后再打开预测、压枪、ESP32 或 Logitech 输出。
4. 每次只改少量参数，观察效果后再继续。

## 常见问题

- 面板打不开：检查 Python 是否安装、依赖是否安装、PySide6 是否安装成功。
- 参数改了没效果：确认已保存，并确认对应参数是否需要重启推理核心。
- 模型编译失败：检查 `.onnx` 是否有效、显卡驱动和 VC++ 运行库是否正常、`engine_builder` 是否完整。
- ESP32 不生效：检查串口号、波特率、固件协议和 `pyserial` 依赖。
- Logitech 输出不生效：检查 `runtime/logi_driver.dll` 是否存在，必要时以管理员权限运行。
